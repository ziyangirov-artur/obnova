## №12247 (12.03.2026) — Исключения пользователей для блокировки по долгу по сделкам

**Суть:** При создании задачи по сделке (долг клиента) проверяется настройка «ПользователиБезБлокировкиСДолгомПоСделкам». Если текущий пользователь входит в список исключений, у задачи выставляется `НеБлокирует1С = Истина`, и блокировка 1С по этой задаче не срабатывает.

**Участок кода (создание новой задачи при наличии долга):**

```bsl
	Если НаборЗадач.Количество() = 0 И СуммаДолга <> 0 Тогда
		// Создаем задачу
		НовЗадача = НаборЗадач.Добавить();
		НовЗадача.БлокировкаПоЗадолженности = Истина;
		НовЗадача.Отсрочка 					= Истина;
		НовЗадача.Объект 					= Сделка;
		НовЗадача.ДатаВремяОтсрочки			= ТекущаяДата() + ДнейОтсрочки * СуткиВСекундах;
		НовЗадача.Пользователь 				= пПользователь;
		НовЗадача.Тема 						= пТема;
		НовЗадача.Описание 					= пОписание;
		//+Бардашов Н.А. 19.02.2026 №12247  
		ПользователиИсключения = РегистрыСведений.АвтосетьНастройки.ПолучитьПараметрыСписком("ПользователиБезБлокировкиСДолгомПоСделкам"); 
		Если ПользователиИсключения.Найти(пПользователь) <> Неопределено Тогда 
			НовЗадача.НеБлокирует1С = Истина;
		КонецЕсли;	
		//-Бардашов Н.А. 19.02.2026 №12247
		НаборЗадач.Записать();
```

**Используемые объекты:** регистр сведений `АвтосетьНастройки` (параметр «ПользователиБезБлокировкиСДолгомПоСделкам»), регистр `НеобходимАкцептПоОбъектам` (реквизит `НеБлокирует1С`).

## №11555 (16.03.2026) — Пропуск обновления свойства «Входящий трафик»

**Суть:** В условии пропуска создания/обновления (возврат из процедуры без создания события/задачи) добавлено «Или Свойство = "Входящий трафик"». Для подразделений из настройки «ПодразделениеНеСоздаватьЗадачи» свойство «Входящий трафик» в этом контексте не обновляется (не создаётся новое событие/задача).

**Участок кода:**

```bsl
	//+Бардашов Н.А. 30.01.2026 №11555 
	ПодразделенияИсключения = РегистрыСведений.АвтосетьНастройки.ПолучитьНастройки("ПодразделениеНеСоздаватьЗадачи");  
	Если ПодразделенияИсключения.Количество() <> 0 Тогда 
		ПодразделениеДокумента = ДокументСсылка.ПодразделениеКомпании;
		ЗначениеНастройки = ПодразделенияИсключения.Найти(ПодразделениеДокумента,"Параметр");
		Если ЗначениеНастройки <> Неопределено И (ЗначениеСвойства = "Звонок заботы" 
			Или ЗначениеСвойства = "Рекомендация 1"
			Или ЗначениеСвойства = "Рекомендация 2"
			Или ДопПараметр = "Дружеские отношения" 
			Или Свойство = "Входящий трафик") Тогда //+Бардашов Н.А. 16.03.2026 №11555
			Возврат;
		КонецЕсли;                                           
	КонецЕсли;	
	//-Бардашов Н.А. 30.01.2026 №11555
```

**Используемые объекты:** регистр сведений `АвтосетьНастройки` (настройка «ПодразделениеНеСоздаватьЗадачи»). Контекст: процедура формирования параметров события/задачи по документу (Рабочий лист ОП / Реализация и этапы CRM).

## №13294 (13.03.2026) — ВидСобытия при создании «События» из рабочего листа ОП (телемаркетолог)

**Суть:** При создании документа «Событие» из рабочего листа ОП: если в `СтруктураДанных` есть «Телемаркетолог» (или контакт от телемаркетолога), то `ВидСобытия` берётся из `СтруктураДанных.ВидКонтакта`, иначе — как раньше (ВидКонтакта или ТелефонныйЗвонок). В текущей реализации для Рабочего листа ОП вид события уже задаётся из `СтруктураДанных.ВидКонтакта`.

**Участок кода (присвоение ВидСобытия при создании нового события):**

```bsl
		НовыйДокументСобытие.Дата			 			  = ТекущаяДата();
		НовыйДокументСобытие.Тема			 			  = ?(Свойство = "Входящий трафик", "Повторный контакт по входящему трафику", ЗначениеСвойства);
		НовыйДокументСобытие.Приоритет 	     			  = Перечисления.Важность.Высокая;
		НовыйДокументСобытие.Состояние 	      			  = Перечисления.СостоянияСобытий.Запланировано;
		НовыйДокументСобытие.ТипСообщения      			  = ?(ЭтоРабочийЛистОП, Перечисления.ТипыСообщений.Входящее, Перечисления.ТипыСообщений.Исходящее);
		НовыйДокументСобытие.ВидСобытия        			  = ?(ЭтоРабочийЛистОП, СтруктураДанных.ВидКонтакта, Перечисления.ВидыСобытий.ТелефонныйЗвонок);
		НовыйДокументСобытие.ДатаНачала 	   			  = ?(ЭтоРабочийЛистОП, СтруктураДанных.ДатаПришел, ДатаСобытия);
```

Для ветки с телемаркетологом (проверка при смене менеджера и «Входящий трафик»):

```bsl
			// Дьяконов+ (03.06.24) ORG-509 Если изменился менеджер и вид события личная встреча, то проверим вел клиента телемаркетолог или нет.
			Если ПоследнееСобытие.Менеджер <> ДокументСсылка.Менеджер И НовыйДокументСобытие.ВидСобытия = Перечисления.ВидыСобытий.ЛичнаяВстреча И ЗначениеСвойства = "Входящий трафик" Тогда
				КлиентаВелТелемаркетолог = обПраво("Телемаркетолог",,,,ПоследнееСобытие.Пользователи[0].Пользователь);
				Если КлиентаВелТелемаркетолог Тогда
					ЗначенияСвойствОбъекта = Обработки.ЗначенияСвойствОбъекта.Создать();
					...
					Свойство = ЗначенияСвойствОбъекта.СвойстваИЗначения.Найти(ПланыВидовХарактеристик.СвойстваОбъектов.ПереданТелемаркетологом, "Свойство");
					Если Свойство <> Неопределено Тогда
						Свойство.Значение = Истина;

```

**Используемые объекты:** перечисление `ВидыСобытий`, право «Телемаркетолог», свойство объекта «ПереданТелемаркетологом», обработка `ЗначенияСвойствОбъекта`.

## №12823 (10.03.2026, 16.03.2026) — НикТГ и запись в КонтактнаяИнформация; НикТГ в Рабочем листе ОП

**Суть (по отчёту):**  
- При создании события заполняется реквизит НикТГ; при наличии Контрагента и НикТГ выполняется запись в регистр `КонтактнаяИнформация` (Тип = НикТГ).  
- В создании Рабочего листа ОП добавлено поле НикТГ и передача НикТГ в `СтруктураДанных` при создании события (если нет СсылкаНаЛидLMS).  
- В НовыйРабочийЛистОП добавлено присвоение `НикТГ = СтрокаРегистра.НикТГ`.

**Примечание:** В модуле `Odin_Общий` прямых вхождений «НикТГ» или типа «НикТГ» в `КонтактнаяИнформация` не найдено. Реализация НикТГ и записи в регистр `КонтактнаяИнформация` может находиться в модулях документа **Событие**, **РабочийЛист** или в подписках на события. Ниже — типовой фрагмент работы с регистром `КонтактнаяИнформация` в том же модуле (запись телефона):

```bsl
	ЗаписатьТелефон = РегистрыСведений.КонтактнаяИнформация.СоздатьМенеджерЗаписи();
	ЗаписатьТелефон.Объект = НовыйКонтрагент.Ссылка;
	ЗаписатьТелефон.Вид = Справочники.ВидыКонтактнойИнформации.ТелефонКонтактный;
	ЗаписатьТелефон.Тип = Перечисления.ТипыКонтактнойИнформации.Телефон;
	...
```

Для полного разбора №12823 целесообразно просмотреть объектные модули документов **Событие** и **РабочийЛист**, подписки на события этих документов и процедуры создания Рабочего листа ОП (например, `СоздатьОбновитьРабочийЛистОП`, `НовыйРабочийЛистОП`).

## Бардашов (без номера) — Событие по тест-драйву: состояние Запланировано вместо Завершено

**Суть:** Событие, создаваемое по тест-драйву, должно создаваться со состоянием **Запланировано**, а не **Завершено**, чтобы отражать предстоящий тест-драйв, а не уже состоявшийся.

**Текущий участок кода (создание события по тест-драйву):**

```bsl
	ИначеЕсли ДопПараметр = "Тест драйв" Тогда
		СобытиеПоТестДрайву = Документы.Событие.СоздатьДокумент();
		СобытиеПоТестДрайву.ДокументОснование = ДокументСсылка;
		СобытиеПоТестДрайву.Заполнить(СобытиеПоТестДрайву.ДокументОснование);
		
		СобытиеПоТестДрайву.Дата			 			 = ТекущаяДата();
		СобытиеПоТестДрайву.Тема			   			 = "Тест драйв";
		СобытиеПоТестДрайву.ДатаНачала 	   				 = СтруктураДанных.ДатаПришел;
		СобытиеПоТестДрайву.ДатаОкончания				 = СтруктураДанных.ДатаУшел;
		СобытиеПоТестДрайву.Контрагент    				 = СтруктураДанных.Контрагент;
		СобытиеПоТестДрайву.КонтактноеЛицо				 = СтруктураДанных.Контрагент;
		СобытиеПоТестДрайву.Приоритет					 = Перечисления.Важность.Высокая;
		СобытиеПоТестДрайву.ТипСообщения      			 = Перечисления.ТипыСообщений.Входящее;
		СобытиеПоТестДрайву.Состояние 	      			 = Перечисления.СостоянияСобытий.Завершено;
		СобытиеПоТестДрайву.ВидСобытия					 = Перечисления.ВидыСобытий.ЛичнаяВстреча;
		...
```

**Рекомендуемое изменение (по описанию правки):** заменить присвоение состояния с `Завершено` на `Запланировано`:

```bsl
		СобытиеПоТестДрайву.Состояние 	      			 = Перечисления.СостоянияСобытий.Запланировано;
```

То есть для сценария «тест-драйв запланирован» использовать `СостоянияСобытий.Запланировано` вместо `СостоянияСобытий.Завершено`.
